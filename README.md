# Dungeon Challenge Processor
## Описание проекта
Сервис для обработки игровых событий.
Приложение читает конфигурацию подземелья и лог событий, симулирует состояние игры, валидирует действия игроков и генерирует итоговый отчет в заданном формате.

## Технологии
- **Язык**: Go 1.25
- **Конфиг**: github.com/ilyakaznacheev/cleanenv
- **Тесты**: testify

### Запуск

```bash
go run cmd/main.go

# или

make run
```

### Запуск юнит-тестов

```bash
make test
```

## Переменные окружения
- `DUNGEON_CFG_PATH` - путь к файлу конфигурации подземелья. default: `config.json`
- `EVENTS_PATH` - путь к файлу с событиями. default: `events`

## Конфигурация подземелья
Система поддерживает и валидирует следующий диапазон значений:
- 1 <= Floors <= N
- 0 <= Monsters <= N
- 00:00:00 <= OpenAt <= CloseTime <= 23:59:59

**Допущения сделанные на основе example логов**:
1. **Дисквалифицированные игроки**: Все события от игрока после его дисквалификации игнорируются (в логах не отображаются)
2. **Расположение босса**: Босс всегда находится на последнем этаже
3. **Вход к боссу**: Событие "entered the boss's floor" (ID 6) валидно только в том случае, если игрок предварительно поднялся на последний этаж
4. **Дисквалификация**: наступает только в случае попытки зайти в подземелье без регистрации и не возможности продолжать (ID 9).
Во всех остальных неправомерных случаях выводится лог "makes imposible move"

**Дополнительные авторские допущения (не описанные в правилах)**:
1. **Закрытие подземелья**: Если timestamp события превышает время закрытия (CloseTime), текущее и все последующие события не обрабатываются. 
При закрытии подземелья - игроки, оставшиеся в подземелье, принудительно покидают его с фиксацией общего времени TotalTime.
2. **Повторный визит на этаж**: Если игрок возвращается на уже посещенный этаж, время входа не обновляется
3. **При восстановлении HP или получении урона**, если ExtraParam не валиден (отрицательный либо не может конвертнуться в число) - выводится лог "makes imposible move"
4. При значении Monsters = 0 в конфиге, все этажи (кроме последнего) считаются уже зачищенными 
5. При значении Floors = 1 в конфиге, игрок при входе в подземелье уже находится на этаже с боссом (ему остается только вызвать событие 6)
